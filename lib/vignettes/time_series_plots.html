Recreating and Modifying Time Series Plots

n.shephard@sheffield.ac.uk

2018-03-28

Overview

A number of time-series plots were created for this work in the statistical softawre R using the ggplot2 package. A selection are shown below. This document seeks to explain how these were created and how to go about recreating, modifying or adapting the plots.

Installation

There are a number of requirements that need to be fulfilled before you can start working with the graphs…

  1. R installed.
  2. RSutdio installed (an Integrated Development Environment (IDE), not essential but highly recommended).

  3. The closed package that has been written for this work must be installed. It is hosted on GitHub and therefore requires the devtools package to install it. In RStudio just type…

    install.packages(“devtools”) install_github(“ns-ctru/closed”)

If after having installed this package you see at any stage messages along the lines of there is no package called 'xxxx' then please contact the package author who will add it as a dependency to the closed package. After that has done repeat step 3 and reinstall the package and it should pull in the missing package. You can however also manually install the missing package yourself with…

install.packages("xxx")

Usage

The closed includes a number of functions that simplify the process of doing the same task (whether thats plotting, tabulating, regression models) repeatedly across the multitude of outcomes of interest (some 54!). Of particular relevance to recreating graphs are the closed_ts_plot() and closed_ts_plot_binary() functions which generate the time-series plots. Examples of the graphs that can be generated are shown in-line below with the underlying function call along with the resulting plot. These can be used as a basis for developing or modifying the plots that are required. Knowledge of the datasets and variables (in terms of the values the variables measure and submeasure take in each dataset) are required and are detailed in the packages README.

Functions are written as it makes maintaining and modifying code a lot easier, you only have to change the underlying code in one place (the function) rather than 54 places if you wish to make the same change to all plots.

NB : Note that closed_ts_plot_lsoa_binary() takes LSOA level data files and internally derives the dichotomised High/Low summary of LSOAs that are above/below the median for the given outcome. As such it can not be used with the data files on the network drive as LSOA level data itself is not available anywhere other than the Virtual Machine. Instead the summary data frames where the data has already been dichotomised into High/Low can be passed to the closed_ts_plot() function described below.

closed_ts_plot()

The closed_ts_plot() function is documented in its help page which can be accessed (after having installed the closed package as described above) by typing ?closed::closed_ts_plot in RStudio. The options are however described below….

Option Default Description

sites c("Bishop Auckland", "Hartlepool", "Hemel Hempstead", "Newark", "Rochdale") The sites to plot.

From Scratch

The function closed_ts_plot() is simply a wrapper that takes various arguments and conditional on these adds or removes features from the plot, it also includes code which defines the various steps that are included for each site, simplifying this having to be explicitly defined each time a graph is generated. The graphs can be created from scratching using a little bit of filtering (using dplyr) and ggplot2. This examples below show how to do this (the magrittr pipe %>% is used extensively and makes code much more readable) and do so in a manner that demonstrates how ggplot2 works which in essence adds (or removes depending on the specification) features to the plot in steps (read the code and the comments which are lines beginning with ## for more information).

## Load required libraries
library(ggplot2)
## Filter the current data frame for the sites of interest
df %>%
    dplyr::filter(town %in% c("Bishop Auckland", "Hartlepool", "Hemel Hempstead", "Newark", "Rochdale")) %>%
## Filter the subset of sites for a single measure/sub.measure
    dplyr::filter(measure == "ed attendances" & sub.measure == "ambulance") %>%
## Start building the plot, first we define what we want to plot on x and y and
## how we are to colour them
    ggplot(aes(x = relative.month,
               y = value,
               colour = town)) +
## Add a geom, in this case its lines
    geom_line()
